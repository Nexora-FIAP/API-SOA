# Política de Segurança

| Versão         | Suporte de Segurança |
|----------------|----------------------|

---

## 🔐 Como Reportar uma Vulnerabilidade

Se você encontrou uma vulnerabilidade de segurança neste projeto, pedimos que siga o processo de **divulgação responsável**:

1. **Não abra uma issue pública** com detalhes da vulnerabilidade.
2. Envie um e-mail para: **[leticiacgresina@gmail.com.com]**
   - Se preferir, abra um **Security Advisory privado** aqui no GitHub (disponível em repositórios públicos).
3. Inclua na sua mensagem:
   - Descrição do problema
   - Etapas para reproduzir (se aplicável)
   - Impacto estimado
   - Recomendação (se tiver)
4. Nós nos comprometemos a:
   - Responder em até **3 dias úteis**
   - Avaliar e confirmar o problema
   - Trabalhar em uma correção
   - Coordenar a divulgação pública (se necessário)

Sua contribuição para manter este projeto seguro é muito importante — obrigado por ajudar!

## 📄 Informações Adicionais

- Este projeto utiliza ferramentas de análise automatizada de segurança como:
  - SAST (Semgrep)
  - DAST (OWASP ZAP)
  - SCA (OWASP Dependency-Check, Dependabot)

Esses mecanismos ajudam a identificar vulnerabilidades desde o desenvolvimento até a entrega da aplicação.
